# Chromashift — готовая WebGL/PWA-сборка

В репозитории находятся две одинаковые копии сайта:

- `site/` — для Render;
- `docs/` — для GitHub Pages без GitHub Actions и без дополнительных OAuth-разрешений.

## Уже настроенный адрес GitHub Pages

Для репозитория `kaufmanbora-lang/chamelion` используется ветка `main` и каталог `/docs`. После публикации игра открывается по адресу:

`https://kaufmanbora-lang.github.io/chamelion/`

Файл `WebGL.data.unityweb` больше лимита браузерной формы GitHub в 25 МБ, но меньше лимита обычного Git. Поэтому проект нужно отправлять через Git/GitHub Desktop, а не через кнопку `Upload files`.

## Render

Файл `render.yaml` создаёт бесплатный Static Site из каталога `site/`, проверяет структуру сборки и выставляет необходимые Brotli/MIME-заголовки Unity.

1. На Render выберите `New → Blueprint`.
2. Подключите репозиторий `kaufmanbora-lang/chamelion`.
3. Подтвердите Blueprint.

После этого Render будет автоматически обновлять сайт при каждом новом коммите в `main`.

## Установка на iPhone

Откройте HTTPS-ссылку игры в Safari, нажмите `Поделиться`, затем `На экран «Домой»`. Игра появится отдельной иконкой и будет запускаться в полноэкранном режиме. Интернет нужен для первой загрузки и сетевых матчей.

## Онлайн-лобби

Сборка подключена к Unity Lobby/Relay. Хост создаёт закрытую комнату и передаёт другим игрокам шестизначный код; одновременно поддерживается до шести игроков.

## Важно

- Не переименовывайте файлы внутри `site/Build` и `docs/Build`.
- Не добавляйте `<base href="/">` в `index.html`: это ломает проектный путь GitHub Pages.
- Не включайте Git LFS для `*.unityweb`: GitHub Pages должен получать настоящие игровые файлы, а не LFS-указатели.
- `site/` и `docs/` должны обновляться одной и той же WebGL-сборкой.
